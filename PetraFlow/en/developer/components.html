Component Development#

PetraFlow modular components are small JuMP model writers with explicit metadata.

Minimal Component Shape#

Base.@kwdef struct MyComponent <: AbstractPowerDeviceComponent
    limit::Float64 = 1.0
end

formulation_short_name(::MyComponent) = "My-Component"
formulation_family(::MyComponent) = :device
formulation_problem_class(::MyComponent) = :LP
required_networks(::MyComponent) = [:power]
required_variables(::MyComponent) = [(:power, :Pg)]
provided_variables(::MyComponent) = Tuple{Symbol,Symbol}[]

function register_constraints!(ctx::OptimizationBuildContext, c::MyComponent)
    Pg = get_variable(ctx, :power, :Pg)
    con = JuMP.@constraint(ctx.model, [g = axes(Pg, 1), t = axes(Pg, 2)], Pg[g, t] <= c.limit)
    register_constraint!(ctx, :power, :my_component_limit, con)
    return ctx
end

Build Order#

build_optimization_model! performs:

  1. required-network checks;

  2. variable registration for every component;

  3. required-variable checks;

  4. constraint registration for every component;

  5. objective-term registration;

  6. final objective construction.

Components should not assume they run immediately after a particular other component. Use required_variables and provided_variables to declare the contract.

Problem Class#

Set formulation_problem_class conservatively:

Class

Use when

:LP

Linear constraints and linear objectives

:QP

Convex quadratic objective or quadratic constraints handled by the intended solver

:NLP

Nonlinear expressions or trigonometric AC equations

:SOCP

Second-order or rotated second-order cone constraints

:MILP

Linear constraints with integer variables

:MISOCP

Conic constraints with integer variables

:MINLP

Nonlinear constraints with integer variables

The router combines classes from all added components.

Testing Checklist#

When adding a component:

  • test descriptor metadata through component_descriptor;

  • test variable and constraint registration;

  • test infer_modular_problem_class;

  • compare against a direct model or an external benchmark formulation when practical;

  • document the mathematical formulation in docs/POWER_COMPONENT_FORMULATION_CONSTRAINTS.md;

  • add the component to _implemented_component_types() if it is public.
