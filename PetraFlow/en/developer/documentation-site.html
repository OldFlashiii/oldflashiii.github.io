Documentation Site#

The documentation tree follows a PowerModels-style layout and now supports two build routes:

  • docs/make.jl: the Julia/Documenter.jl route.

  • docs/sphinx: the Sphinx + MyST route for a minimal bilingual site with a language switch and GitHub links.

docs/
  Project.toml
  make.jl
  sphinx/
    conf.py
    requirements.txt
  src/
    index.md
    sphinx-index.md
    zh/
    en/

Local Documenter.jl Build#

julia --project=docs docs/make.jl

The generated site is written to:

docs/build/

Local Sphinx Build#

Install dependencies:

python -m pip install -r docs\sphinx\requirements.txt

Build from the repository root:

python -m sphinx -b html -d docs\sphinx\_build\doctrees -c docs\sphinx docs\src docs\sphinx\_build\html

Sphinx uses docs/src/sphinx-index.md as its home page, so it does not change the Documenter.jl docs/src/index.md page.

Navigation Model#

The site follows a PowerModels-style layout:

  • Manual: workflows and tutorials.

  • Library: API, problem specifications, network formulations, and component catalog.

  • Developer: extension and maintenance guides.

Bilingual Maintenance#

  • Chinese pages live in docs/src/zh/.

  • English pages live in docs/src/en/.

  • The two language trees should keep the same page structure.

  • New public APIs should update both public-api.md pages.

  • New components should update the component catalog and the mathematical formulation manual.
