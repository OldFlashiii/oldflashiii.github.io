ACOPF Formulations#

PetraFlow separates the ACOPF problem from the algebraic formulation used to write it.

Polar ACP#

init_power!(ctx; formulation = :acp, state_variables = :polar)
add_modules!(ctx, ACPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • voltage magnitude V;

  • voltage angle theta;

  • generator powers Pg, Qg;

  • branch powers Pbr_from, Qbr_from, Pbr_to, Qbr_to.

This formulation uses nonlinear trigonometric branch-flow equations. Voltage magnitude bounds are imposed on V and one reference-bus angle is fixed.

Rectangular ACR#

init_power!(ctx; formulation = :acr, state_variables = :rectangular)
add_modules!(ctx, ACRPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • real voltage Vr;

  • imaginary voltage Vi;

  • generator powers;

  • branch powers.

Voltage magnitude limits are written as quadratic inequalities in Vr and Vi. Reference-bus angle is enforced by fixing the imaginary component.

Current-Voltage IVR#

init_power!(ctx; formulation = :ivr, state_variables = :ivr)
add_modules!(ctx, IVRPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • Vr, Vi;

  • branch currents Ir_from, Ii_from, Ir_to, Ii_to;

  • generator powers;

  • branch powers.

The formulation separates Ohm-law current equations from power definitions.

W-Space SOCP#

init_power!(ctx; formulation = :wsoc, state_variables = :w_space)
add_modules!(ctx, WSOCPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • bus voltage products Wii;

  • branch voltage-product real and imaginary parts Wr, Wi;

  • generator powers;

  • branch powers.

The rank-one voltage-product condition is relaxed with rotated second-order cone constraints. This is the formulation used for the current PowerModels SOCWR comparison test.

W-Space SDP#

init_power!(ctx; formulation = :w_sdp, state_variables = :w_sdp)
add_modules!(ctx, WSDPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • W_sdp[t], a real lifted positive-semidefinite voltage-product matrix;

  • generator powers;

  • branch powers.

This formulation uses a real \(2n\times 2n\) symmetric PSD matrix for the lifted complex-voltage product and drops the rank-one condition. It is useful for small SDP-relaxation studies, lower-bound checks, and comparisons against SOC and QC relaxations.

QC Relaxation#

init_power!(ctx; formulation = :qc, state_variables = :w_space)
add_modules!(ctx, QCPowerFlowRelaxation(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • Wii, Wr, and Wi;

  • generator powers;

  • branch powers.

QC relaxations strengthen W-space models with convex envelopes for voltage products, trigonometric terms, and bilinear expressions. The current component keeps a unified QC entry point while reusing the W-space SOC variable layer.

Branch-Flow SOCP#

init_power!(ctx; formulation = :bf_soc, state_variables = :voltage_square)
add_modules!(ctx, BranchFlowSOCPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

Variables:

  • squared voltage magnitude;

  • squared branch current;

  • branch active and reactive power.

This form is useful for radial or distribution-style networks.

Validation#

test/unit/test_modular_acopf_formulations.jl checks:

  • ACP, ACR, and IVR on PGLIB 118 converge to matching objectives with Ipopt;

  • modular ACP matches PowerModels ACP on the same case;

  • modular W-space SOCP is close to PowerModels SOCWR where Gurobi and Ipopt are available;

  • WSDPPowerFlow initializes the PSD matrix, registers SDP cone constraints, and couples them to W-space branch-flow equations.

The mathematical expressions are documented in the repository-level file docs/POWER_COMPONENT_FORMULATION_CONSTRAINTS.md.
