Component Catalog#

Each modular component declares its formulation name, problem class, required variables, provided variables, and network dependencies.

This page is a registry-oriented quick reference. Mathematical constraints and references are documented in Components and Formulations.

Basic Limit Component Inputs#

Component

Common constructor

User inputs

GeneratorLimits

GeneratorLimits([120.0, 80.0], [20.0, 0.0])

pmax, pmin, qmax, qmin, include_reactive, units, mode

VoltageLimits

VoltageLimits(0.95, 1.05)

vmin, vmax, mode

BranchLimits

BranchLimits(80.0, angle_min = -20.0, angle_max = 20.0)

rate, angle_min, angle_max, include_angle_limits, units, angle_units, mode

GenerationCost

GenerationCost([0.01, 20.0, 0.0])

coefficients

With no arguments, these components read defaults from the network file. With user inputs, they can replace defaults using mode = :override or tighten them using mode = :tighten. See Devices, Limits, and Costs for mathematical constraints and full parameter notes.
