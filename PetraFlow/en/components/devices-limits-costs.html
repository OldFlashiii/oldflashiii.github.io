Devices, Limits, and Costs#

This page covers generators, loads, branches, shunts, transformers, and objective components. These components are usually combined with a power-flow formulation:

add_modules!(
    ctx,
    ACPPowerFlow(),
    VoltageLimits(0.95, 1.05),
    GeneratorLimits(),
    BranchLimits(),
    LoadModel(),
    ShuntFixedAC(),
    TransformerFixedTapAC(),
    GenerationCost(),
)

Basic limit components follow one rule: with no arguments they read the network data; with arguments they override those values, or tighten them with mode = :tighten. Scalars apply to all devices, vectors are ordered by device index, matrices are interpreted as device by time, and dictionaries may use local indices, external ids, (index, time), or (external_id, time) keys.

GeneratorLimits#

Variables:

\[ P_{g,t},Q_{g,t} \]

Limits:

\[ P_g^{min}\le P_{g,t}\le P_g^{max} \]
\[ Q_g^{min}\le Q_{g,t}\le Q_g^{max} \]

With unit commitment:

\[ P_g^{min}u_{g,t}\le P_{g,t}\le P_g^{max}u_{g,t} \]

Common calls:

GeneratorLimits()
GeneratorLimits([120.0, 80.0], [20.0, 0.0])
GeneratorLimits(pmax = 120.0, pmin = 10.0, qmax = 50.0, qmin = -50.0)
GeneratorLimits(1.2, 0.2; units = :pu, include_reactive = false)
GeneratorLimits(pmax = Dict(1 => 120.0, (2, 3) => 90.0), pmin = 0.0, mode = :tighten)

Parameter

Meaning

pmax, pmin

Active-power upper/lower bounds; defaults to gen.pmax and gen.pmin

qmax, qmin

Reactive-power upper/lower bounds; used when Qg exists and include_reactive = true

include_reactive

Whether to apply reactive bounds

units

:physical for MW/MVAr, :pu for per-unit user inputs

mode

:override replaces network limits; :tighten intersects with network limits

VoltageLimits#

VoltageLimits() leaves vm_min and vm_max unchanged. Add explicit values when an experiment needs uniform or local voltage-bound changes:

VoltageLimits(0.95, 1.05)
VoltageLimits(vmin = Dict(5 => 0.97), vmax = 1.04, mode = :tighten)

In polar AC models, the component writes variable bounds:

\[ V_i^{min}\le V_{i,t}\le V_i^{max}. \]

In rectangular, IVR, W-SOC, W-SDP, LinDistFlow, and branch-flow formulations, the same parameters are converted to squared-voltage bounds:

\[ (V_i^{min})^2\le v_{i,t}\le (V_i^{max})^2. \]

Parameter

Meaning

vmin, vmax

Voltage magnitude bounds in p.u.; supports scalar, vector, matrix, and dictionary inputs

mode

:override replaces network voltage limits; :tighten tightens them

LoadModel and LoadZIP#

Fixed demand:

\[ P_{i,t}^{load}=P_i^d,\qquad Q_{i,t}^{load}=Q_i^d \]

ZIP demand with \(v_i=|V_i|^2\):

\[ P_i^{ZIP}(V)=P_i^0\left(a_p^Z v_i+a_p^I |V_i|+a_p^P\right) \]
\[ Q_i^{ZIP}(V)=Q_i^0\left(a_q^Z v_i+a_q^I |V_i|+a_q^P\right) \]
\[ a_p^Z+a_p^I+a_p^P=1,\qquad a_q^Z+a_q^I+a_q^P=1 \]

Class: NLP unless only the constant-power term is used.

LoadSheddingLP and LoadSheddingCost#

\[ 0\le p_{i,t}^{shed}\le P_i^d,\qquad 0\le q_{i,t}^{shed}\le Q_i^d \]
\[ P_i^{net-load}=P_i^d-p_{i,t}^{shed} \]
\[ \min \sum_{i,t} c_i^{shed}p_{i,t}^{shed} \]

This is useful for infeasibility diagnosis, SCOPF corrective actions, and flexible demand.

BranchLimits#

Thermal limits:

\[ P_{ij,t}^2+Q_{ij,t}^2\le (S_{ij}^{max})^2 \]
\[ P_{ji,t}^2+Q_{ji,t}^2\le (S_{ji}^{max})^2 \]

Angle difference:

\[ \theta_{ij}^{min}\le \theta_i-\theta_j\le \theta_{ij}^{max} \]

In DC and PTDF models:

\[ -F_{ij}^{max}\le P_{ij,t}\le F_{ij}^{max} \]

Class: LP, QP, NLP, or SOCP depending on the network formulation.

Common calls:

BranchLimits()
BranchLimits(80.0)
BranchLimits(rate = Dict(1 => 80.0, 2 => 100.0), mode = :tighten)
BranchLimits(rate = 0.8; units = :pu)
BranchLimits(rate = 80.0, angle_min = -20.0, angle_max = 20.0)
BranchLimits(rate = 80.0, angle_min = -0.35, angle_max = 0.35; angle_units = :rad)

Parameter

Meaning

rate

Branch rating; defaults to branch.rate_a, with 0 meaning no rating in the network file

angle_min, angle_max

Lower/upper angle-difference limits

include_angle_limits

Whether angle-difference constraints are added

units

:physical for MVA, :pu for per-unit user inputs

angle_units

:deg or :rad

mode

:override replaces network limits; :tighten intersects with network limits

LinePiAC#

Series admittance:

\[ y_{ij}=\frac{1}{r_{ij}+jx_{ij}}=g_{ij}+jb_{ij} \]

With charging susceptance \(b_{ij}^c\) and transformer tap \(\tau=t e^{j\phi}\):

\[ Y_{ff}=\frac{y_{ij}+j b_{ij}^c/2}{|\tau|^2},\qquad Y_{ft}=-\frac{y_{ij}}{\tau^*} \]
\[ Y_{tf}=-\frac{y_{ij}}{\tau},\qquad Y_{tt}=y_{ij}+j b_{ij}^c/2 \]

These admittance blocks feed AC power-flow components and make_ybus.

ShuntFixedAC and variable shunts#

Fixed shunt admittance:

\[ Y_i^{sh}=G_i^{sh}+jB_i^{sh} \]

Power injection can be written as:

\[ P_i^{sh}=G_i^{sh}|V_i|^2,\qquad Q_i^{sh}=-B_i^{sh}|V_i|^2 \]

The sign convention follows the MATPOWER/PetraFlow bus shunt convention for GS and BS. A continuous shunt or SVC susceptance model is:

\[ B_i^{min}\le B_{i,t}^{var}\le B_i^{max} \]
\[ Q_{i,t}^{var}=-B_{i,t}^{var}|V_{i,t}|^2 \]

TransformerFixedTapAC#

A fixed-tap transformer uses the same branch admittance equations as LinePiAC, but \(\tau=t e^{j\phi}\) is usually not one. For adjustable taps:

\[ t^{min}\le t_t\le t^{max} \]

For one-hot OLTC taps:

\[ \sum_{m\in\mathcal{M}} z_{m,t}=1,\qquad t_t=\sum_{m\in\mathcal{M}} t_m z_{m,t},\qquad z_{m,t}\in\{0,1\} \]

GenerationCost and GeneratorPWLCost#

Quadratic cost:

\[ C_g(P_g)=c_{2,g}P_g^2+c_{1,g}P_g+c_{0,g} \]

By default, GenerationCost() reads gencost. Users can pass polynomial coefficients directly:

GenerationCost([0.01, 20.0, 0.0])
GenerationCost(Dict(1 => [0.01, 20.0, 0.0], 2 => [0.02, 18.0, 0.0]))
GenerationCost((c2 = 0.01, c1 = 20.0, c0 = 0.0))

Parameter

Meaning

coefficients

Polynomial coefficients; a scalar is treated as a linear marginal cost and a vector as [c2, c1, c0]

Polynomial cost:

\[ C_g(P_g)=\sum_{k=0}^{K} c_{k,g}P_g^k \]

Piecewise-linear convex-combination form:

\[ P_g=\sum_{m\in\mathcal{M}} \lambda_{g,m}p_{g,m},\qquad C_g=\sum_{m\in\mathcal{M}} \lambda_{g,m}c_{g,m} \]
\[ \sum_{m\in\mathcal{M}}\lambda_{g,m}=1,\qquad \lambda_{g,m}\ge0 \]

SOS2 or binary variables can enforce adjacent segments.

BranchOverloadPenalty#

Soft thermal limit:

\[ P_{ij,t}^2+Q_{ij,t}^2\le (S_{ij}^{max}+s_{ij,t})^2,\qquad s_{ij,t}\ge0 \]

Penalty:

\[ \min \sum_{(i,j),t} c_{ij}^{over}s_{ij,t} \]

For linear DC models:

\[ -F_{ij}^{max}-s_{ij,t}^- \le P_{ij,t}\le F_{ij}^{max}+s_{ij,t}^+ \]

References#

  • The standard OPF objective, generator limits, and network operating constraints trace back to Carpentier, 1962.

  • Economic dispatch, thermal limits, and piecewise-linear cost modeling can be found in Wood, Wollenberg, and Sheble, Power Generation, Operation, and Control, 2013.

  • FACTS and controllable reactive devices: Hingorani and Gyugyi, 2000.
