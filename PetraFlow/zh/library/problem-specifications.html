问题规格#

问题规格描述“要解什么问题”。它不直接规定使用极坐标、直角坐标还是 W-space，而是说明需要哪些物理约束、设备约束和目标函数。

ACOPF#

最小 ACOPF：

init_power!(ctx; formulation = :acp, state_variables = :polar)
add_modules!(ctx, ACPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

在同一问题规格下可以替换 formulation：

add_modules!(ctx, ACRPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())
add_modules!(ctx, IVRPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

DCOPF#

角度形式：

init_power!(ctx; formulation = :dcp, state_variables = :dc)
add_modules!(ctx, DCPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

PTDF 形式：

init_power!(ctx; formulation = :ptdf)
add_modules!(ctx, PTDFPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

凸松弛 OPF#

W-space SDP 松弛：

init_power!(ctx; formulation = :w_sdp, state_variables = :w_sdp)
add_modules!(ctx, WSDPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

W-space SOCP 或 QC 松弛：

init_power!(ctx; formulation = :wsoc, state_variables = :w_space)
add_modules!(ctx, WSOCPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

init_power!(ctx; formulation = :qc, state_variables = :w_space)
add_modules!(ctx, QCPowerFlowRelaxation(), GeneratorLimits(), BranchLimits(), GenerationCost())

AC/DC Hybrid OPF#

init_power!(
    ctx;
    formulation = :acp,
    state_variables = :polar,
    network_type = :acdc,
    hvdc_case = hvdc_case,
)

add_modules!(
    ctx,
    ACPPowerFlow(),
    GeneratorLimits(),
    BranchLimits(),
    DCBusBalance(),
    DCLineSOC(lines = hvdc_dc_line_specs(hvdc_case)),
    VSCQuadraticLossNLP(converters = hvdc_vsc_converter_specs(hvdc_case)),
    GenerationCost(),
)

与 network formulation 的关系#

PetraFlow 将 problem specification 和 network formulation 解耦：问题规格由组件组合表达，network formulation 由 init_power! 和潮流组件表达。这样同一个 OPF 问题可以在 ACP、ACR、IVR、W-space 或 branch-flow 等不同变量空间中复用设备、成本和安全约束组件。
