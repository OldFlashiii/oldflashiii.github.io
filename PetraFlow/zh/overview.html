项目总览#

PetraFlow 是一个面向电力系统稳态分析、最优潮流、模块化优化建模和数值优化算法研究的 Julia 平台。它既保留 runpf、runopf 这类直接入口，也提供 power_context、init_power!、add_modules!、solve_modular_model! 这类组件化建模接口。

PetraFlow 解决什么问题#

PetraFlow 当前围绕三条主线组织：

主线

目标

典型入口

潮流与稳态分析

快速读取算例、计算潮流、构造 Ybus、PTDF、LODF、CPF、N-1 等分析量

load_network, runpf, make_ybus, make_ptdf, static_security_analysis

最优潮流求解平台

对 ACOPF、DCOPF-QP、benchmark 和结构化求解器做统一实验

runopf, solve_opf, build_dcopf_qp, run_benchmark

模块化建模

像搭积木一样组合网络 formulation、设备、成本、限值和求解器

power_context, init_power!, add_modules!, solve_modular_model!

用户工作流#

最常见的流程是：

  1. 读取或构造网络：load_power_network / build_power_network。

  2. 初始化状态变量：选择 :polar、:rectangular、:ivr、:voltage_square、:w_space 或 :dc。

  3. 选择组件：例如 ACPPowerFlow()、GeneratorLimits()、BranchLimits()、GenerationCost()。

  4. 选择求解器：例如 solver = :ipopt、:gurobi、:highs 或 :petraflow。

  5. 查询结果：使用 bus_solution、generator_solution、branch_solution、solution_value。

核心对象#

对象

作用

Network

PetraFlow 的 typed 电力网络数据结构，兼容 MATPOWER/PGLIB 字段

PowerNetwork

模块化优化使用的电力网络 bundle

OptimizationBuildContext

保存 JuMP 模型、网络、时间维度、变量/约束/目标注册表

AbstractOptimizationComponent

所有模块化组件的共同协议

ModularOptimizationResult

模块化模型求解后的统一结果对象
