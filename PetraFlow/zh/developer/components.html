组件开发#

本页说明如何在 PetraFlow 中新增模块化优化组件。

组件最小接口#

Base.@kwdef struct MyLimit <: AbstractPowerDeviceComponent
    limit::Float64 = 1.0
end

formulation_short_name(::MyLimit) = "My-Limit"
formulation_family(::MyLimit) = :device
formulation_problem_class(::MyLimit) = :LP
required_networks(::MyLimit) = [:power]
required_variables(::MyLimit) = [(:power, :Pg)]
provided_variables(::MyLimit) = Tuple{Symbol,Symbol}[]

function register_constraints!(ctx::OptimizationBuildContext, component::MyLimit)
    Pg = get_variable(ctx, :power, :Pg)
    con = JuMP.@constraint(
        ctx.model,
        [g = axes(Pg, 1), t = axes(Pg, 2)],
        Pg[g, t] <= component.limit,
    )
    register_constraint!(ctx, :power, :my_limit, con)
    return ctx
end

开发步骤#

  1. 选择组件类别和抽象父类。

  2. 定义参数 struct。

  3. 声明 formulation、problem class、网络依赖、变量依赖。

  4. 实现 register_variables!，如果组件提供新变量。

  5. 实现 register_constraints!。

  6. 实现 register_objective!，如果组件贡献目标项。

  7. 将用户可见组件加入 _implemented_component_types()。

  8. 添加单元测试。

  9. 在 docs/POWER_COMPONENT_FORMULATION_CONSTRAINTS.md 写数学约束。

问题类型选择#

类型

使用场景

:LP

线性约束和线性目标

:QP

二次目标或二次约束

:NLP

非线性函数、三角函数或非凸约束

:SOCP

二阶锥或旋转二阶锥

:MILP

线性整数模型

:MISOCP

整数 + 锥约束

:MINLP

整数 + 非线性约束

测试建议#

组件测试至少覆盖：

  • descriptor 是否正确；

  • required/provided variables 是否正确；

  • 注册变量的维度；

  • 注册约束数量和类型；

  • infer_modular_problem_class 是否正确；

  • 小算例上是否能求解；

  • 与直接 JuMP 模型或 PowerModels 模型的数值对照。
