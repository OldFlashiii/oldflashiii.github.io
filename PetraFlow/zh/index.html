PetraFlow 中文文档#

PetraFlow 是面向电力系统稳态分析、最优潮流、模块化优化建模和数值优化算法研究的 Julia 平台。

文档采用类似 PowerModels.jl 的组织方式，将用户工作流、组件 formulation、模型库和开发者接口分开：

  • Manual：面向用户的使用手册，按“读取数据、分析网络、构建模型、选择求解器”的顺序展开。

  • Components：按潮流、设备、储能、HVDC/FACTS、安全约束等类型介绍数学公式、变量、参数和文献依据。

  • Library：面向 API 和模型目录，说明 problem specification、network formulation、组件目录和公共函数。

  • Developer：面向开发者，说明如何新增组件、如何写测试、如何维护文档网站。

推荐阅读路径#

  1. 快速开始：运行第一个潮流和模块化 ACOPF。

  2. 项目架构：先看 PetraFlow 的数据层、稳态分析层、模块化建模层和求解器层。

  3. 建模与求解总览：从任务角度理解潮流、OPF 求解平台和模块化建模三条主线。

  4. 网络数据：理解 MATPOWER/PGLIB 字段如何进入 typed Network。

  5. 稳态分析：使用 Ybus、Jacobian、PTDF、LODF、CPF 和 N-1。

  6. 模块化优化：使用 power_context、init_power!、add_modules!。

  7. 组件与 formulation 总览：按元件类型查找公式、参数和文献。

  8. 潮流 formulation：比较 ACP、ACR、IVR、DCP、PTDF、W-SOC、BF-SOC。

  9. AC/DC 与 HVDC：使用 PGLIB-HVDC 和 network_type = :acdc。

  10. 组件目录：查看已实现和规划中的模块化组件。

核心理念#

PetraFlow 区分三件事：

  • 问题规格：要解什么问题，例如 PF、OPF、DCOPF、SCOPF、储能调度、AC/DC 混联 OPF。

  • 网络 formulation：用什么变量和方程表示网络，例如极坐标、直角坐标、IVR、W-space、branch-flow、PTDF。

  • 建模组件：将设备、限值、成本、安全约束和耦合关系加入 JuMP 模型。

这种组织方式可以让同一个问题规格在不同 formulation 下比较，也可以让同一个 formulation 搭配不同设备组件。

文档层级#

层级

内容

Manual

面向用户的流程：读数据、建网络、跑潮流、建 OPF、换求解器、查结果

Components

面向建模的数学细节：潮流、设备、储能、HVDC、FACTS、安全约束和耦合公式

Library

面向 API 的参考：公共函数、问题规格、网络 formulation、组件目录

Developer

面向贡献者：新增组件、维护注册表、测试和文档网站

当前重点#

  • 稳态分析中支持 MATPOWER 风格变压器支路和 bus shunt。

  • 模块化 ACOPF 支持极坐标、直角坐标和 IVR 三种 NLP formulation。

  • 模块化 W-space SOCP 与 PowerModels SOCWR 进行数值对照。

  • AC/DC 混联网络通过 network_type = :acdc 显式初始化 DC 侧变量。

  • PGLIB-HVDC/MatACDC 数据可用于 HVDC 组件测试。
