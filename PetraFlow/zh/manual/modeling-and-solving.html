建模与求解总览#

本页解释 PetraFlow 如何把电力系统数据、稳态分析、OPF 求解平台和模块化建模组织在一起。它面向第一次阅读项目源码或准备写 notebook 的用户：先理解“应该调用什么”，再理解“底层如何拆分”。

PetraFlow 参考 PowerModels.jl 的文档分层思想，将文档分为 Manual、Library 和 Developer。不同之处在于，PetraFlow 同时强调三条主线：

  1. 潮流与稳态分析：不构造优化模型，直接分析网络状态、矩阵和安全裕度。

  2. 最优潮流与求解平台算法：提供 runopf / solve_opf 一行式入口，以及 PetraFlow 内部 NLP/QP 求解器研究接口。

  3. 模块化优化建模：把 JuMP 模型拆成网络初始化、状态变量空间、组件、求解器路由和结果查询。

方向一：潮流与稳态分析#

稳态分析层不需要 JuMP 模型。它直接基于 typed Network 计算电压、支路潮流、导纳矩阵、Jacobian、灵敏度和安全分析结果。

函数

按外部 ID 查询元件

MATPOWER 中的变压器和 shunt 已进入 typed 网络：

  • Branch.tap 和 Branch.shift 表示 off-nominal tap ratio 与 phase shifter。

  • Bus.gs 和 Bus.bs 表示固定有功/无功 shunt。

  • make_ybus 会同时考虑 series admittance、line charging、tap、shift 和 bus shunt。

net = load_network("case118")
pf = solve_pf(net; method = :NR, verbose = false)

调用参数

说明

Newton-Raphson

默认推荐，收敛快，适合大多数 AC PF

使用解耦近似，适合快速近似

method = :FDBX

另一种 fast-decoupled 变体

Gauss-Seidel

method = :GS

适合教学和小算例

HELM

method = :HELM

全纯嵌入潮流路线，用于与迭代法对照

无功限值处理：

pf = solve_pf(net; method = :NR, enforce_q_lims = true)

启用后，PV 母线可能在无功越限时转为 PQ 母线，结果中会记录 q_lim_enforced 和 pv_to_pq_buses。

方向二：最优潮流与求解平台算法#

OPF 路线分为应用层和求解器研究层。普通用户优先使用 runopf / solve_opf；需要研究 KKT、QP_fast 或内点法时，再进入 NLPProblem、QPProblem 和 src/solver。

flowchart TB runopf["runopf / solve_opf"] acopf["ACOPF\nACP / ACR / IVR"] dcopf["DCOPF-QP\nB-theta / PTDF"] external["外部求解器\nIpopt / Gurobi / HiGHS"] internal["PetraFlow 求解器\nPDIPM / QP IPM / KKT route"] report["report_solution / collect_solver_metrics / logs"] runopf --> acopf runopf --> dcopf acopf --> external acopf --> internal dcopf --> external dcopf --> internal external --> report internal --> report

一行式 OPF#

result = runopf(
    "case118";
    formulation = :acp,
    solver = :petraflow,
    acopf_mode = :auto,
    acceleration_mode = :safe,
    verbose = false,
)

方向三：模块化优化建模#

模块化建模层的目标是让用户像搭积木一样构造模型，而不是在 notebook 中手写所有变量和约束。

flowchart TB network["load_power_network / build_power_network"] ctx["power_context\n保存 JuMP.Model + networks + registry"] init["init_power!\n选择 formulation 和 state_variables"] comp["add_modules!\nPowerFlow + Limits + Cost + Devices + Coupling"] build["build_modular_model!\n注册变量 -> 约束 -> 目标"] solve["solve_modular_model!\nsolver routing + optimize!"] inspect["solution_status\nsolution_value\nbus_solution / branch_solution"] network --> ctx --> init --> comp --> build --> solve --> inspect

用户工作流#

power = load_power_network("case14")
ctx = power_context(power; T = 1)

init_power!(ctx; formulation = :acp, state_variables = :polar)
add_modules!(ctx, ACPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

solution = solve_modular_model!(
    ctx;
    solver = :ipopt,
    attributes = Dict("print_level" => 0),
)
