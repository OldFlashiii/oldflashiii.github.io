快速开始#

本页用一条完整路线展示 PetraFlow 的日常使用方式：读取网络、运行潮流、构造一个可解释的小网络、搭建模块化 ACOPF、切换 formulation、查询求解结果。每一步都尽量使用用户层 API，避免直接操作内部数据结构。

1. 加载 PetraFlow#

在项目根目录启动 Julia 后，先激活环境并加载包：

using Pkg
Pkg.activate(".")

using PetraFlow

如果是第一次使用，建议先阅读 安装与环境，确认 Julia 环境和求解器后端已经准备好。

2. 读取一个标准算例#

PetraFlow 可以读取内置 MATPOWER/PGLIB 算例，也可以读取用户自己的 .m 文件。最小例子是 IEEE 14 节点：

net = load_network("case14")

net 是 typed Network，其中母线、发电机、支路、储能和成本都被转换为 Julia 结构体。可以先查看网络规模：

nbus(net)
ngen(net)
nbranch(net)
total_load(net)
total_generation(net)

这一步的目的不是优化，而是确认数据已经被正确解析。对于新算例，建议先检查网络规模、负荷总量、发电机数量、变压器数量和是否存在孤岛。

3. 运行第一个潮流#

潮流计算是最基础的稳态分析。给定发电、负荷和电压设定后，PetraFlow 可以用 Newton-Raphson 等方法求解电压和支路潮流：

pf = solve_pf(net; method = :NR, verbose = false)

pf.success
pf.iterations
pf.max_mismatch

如果 pf.success 为 true，说明潮流收敛。接下来可以使用语义化函数查看结果：

voltage(pf)
generation(pf)
branch_flow(pf)
losses(pf)

这些函数适合 notebook 展示和调试。更复杂的稳态分析可以继续使用：

make_ybus(net)
make_ptdf(net)
make_lodf(net)
static_security_analysis(net)

4. 构造一个小网络#

读取标准算例适合 benchmark；但在教学、测试或验证新组件时，一个小而可解释的网络更有用。PetraFlow 提供 builder API，可以逐项添加元件：

power = build_power_network(name = "two_bus", base_mva = 100.0) do b
    add_bus!(b, 1; type = :ref, vm = 1.0)
    add_bus!(b, 2; type = :pq, pd = 80.0, qd = 20.0)

    add_generator!(
        b,
        1;
        pg = 80.0,
        qg = 20.0,
        pmax = 150.0,
        pmin = 0.0,
        qmax = 80.0,
        qmin = -80.0,
    )

    add_transformer!(b, 1, 2; r = 0.01, x = 0.05, tap = 1.05, shift = 3.0)
    add_capacitor!(b, 2; q_mvar = 5.0)
    add_reactor!(b, 2; q_mvar = 1.0)
end

这里 add_transformer! 会创建带变比和相移的支路；add_capacitor! 和 add_reactor! 会写入母线 shunt。构造完成后，仍然可以回到 typed Network：

net2 = power.data
make_ybus(net2)
count_transformers(net2)
total_shunt(net2)

这种小系统常用于验证“某个设备参数是否被正确进入 Ybus 或优化模型”。

5. 运行模块化 ACOPF#

模块化建模从 PowerNetwork 开始。下面例子读取 PGLIB 118 节点，初始化极坐标 ACOPF，加入潮流、发电机限值、支路限值和发电成本：

power = load_power_network(joinpath(PETRAFLOW_DATA_DIR, "PGLIB", "pglib_opf_case118_ieee.m"))
ctx = power_context(power; T = 1)

init_power!(ctx; formulation = :acp, state_variables = :polar)

add_modules!(
    ctx,
    ACPPowerFlow(),
    GeneratorLimits(),
    BranchLimits(),
    GenerationCost(),
)

这几行代码对应完整 ACOPF 的基本结构：

步骤

作用

power_context

创建优化上下文，内部保存 JuMP 模型、网络和注册表

init_power!

根据 formulation 和状态变量空间创建基础变量

ACPPowerFlow()

写入 AC 极坐标潮流平衡和支路潮流关系

GeneratorLimits()

写入发电机有功/无功上下限

VoltageLimits(0.95, 1.05)

覆盖或收紧母线电压幅值上下限

BranchLimits()

写入支路容量约束

GenerationCost()

写入发电成本目标项

限值和成本组件可以直接接收用户参数。例如：

add_modules!(
    ctx,
    ACPPowerFlow(),
    VoltageLimits(0.95, 1.05),
    GeneratorLimits([120.0, 80.0], [20.0, 0.0]),
    BranchLimits(80.0, angle_min = -20.0, angle_max = 20.0),
    GenerationCost([0.01, 20.0, 0.0]),
)

求解模型：

result = solve_modular_model!(
    ctx;
    solver = :ipopt,
    attributes = Dict("print_level" => 0, "tol" => 1.0e-7),
)

6. 查询求解结果#

求解后可以先看状态：

solution_status(result)
result.termination_status
result.primal_status
result.selection

再按电力系统语义查询：

bus_solution(result; time = 1)
generator_solution(result; time = 1)
branch_solution(result; time = 1)

也可以按变量名精确查询某个变量：

available_solution_variables(result)
solution_value(result, :power, :vm; index = 1, time = 1)
solution_value(result, :power, :pg; index = 1, time = 1)

这种查询方式是模块化建模的重要部分：用户可以按 bus、generator、branch、storage 等对象理解结果，而不必记住 JuMP 内部变量容器的具体索引。

7. 切换 formulation#

同一个网络和同一组设备组件，可以在不同状态变量空间中比较。极坐标 ACOPF：

ctx = power_context(power; T = 1)
init_power!(ctx; formulation = :acp, state_variables = :polar)
add_modules!(ctx, ACPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

直角坐标 ACOPF：

ctx = power_context(power; T = 1)
init_power!(ctx; formulation = :acr, state_variables = :rectangular)
add_modules!(ctx, ACRPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

IVR 电流-电压 formulation：

ctx = power_context(power; T = 1)
init_power!(ctx; formulation = :ivr, state_variables = :ivr)
add_modules!(ctx, IVRPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

凸松弛模型也遵循同样的组合方式。例如 W-space SOCP：

ctx = power_context(power; T = 1)
init_power!(ctx; formulation = :w_soc, state_variables = :w_space)
add_modules!(ctx, WSOCPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

8. 查询可用组件#

如果不确定当前项目已经实现了哪些组件，可以从注册表开始：

registered_optimization_modules()
find_optimization_modules(:power_flow)
find_optimization_modules(:storage)
find_optimization_modules(:hvdc)
module_coverage_report()

注册表会展示组件名称、formulation、问题类别、依赖网络、依赖变量和实现状态。它是从“我想加入某类设备”走向“我应该调用哪个组件”的最直接入口。
