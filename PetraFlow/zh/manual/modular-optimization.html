模块化优化#

模块化优化层把一个 JuMP 模型拆成网络初始化和多个组件。每个组件只负责自己的变量、约束和目标项，用户通过组合组件构造问题。

五步工作流#

步骤

函数

作用

1

load_power_network / build_power_network

准备网络

2

power_context / modular_context

创建上下文

3

init_power!

初始化变量空间

4

add_modules!

添加组件

5

solve_modular_model!

构建、配置求解器并求解

 
power = load_power_network("case14")
ctx = power_context(power; T = 1)

init_power!(ctx; formulation = :dcp, state_variables = :dc)
add_modules!(ctx, DCPPowerFlow(), GeneratorLimits(), BranchLimits(), GenerationCost())

result = solve_modular_model!(ctx; solver = :gurobi)

上下文#

OptimizationBuildContext 保存：

  • ctx.model：JuMP 模型。

  • ctx.networks：SystemNetworkBundle。

  • ctx.variables：变量注册表。

  • ctx.constraints：约束注册表。

  • ctx.objective_terms：目标项列表。

  • ctx.components：已添加组件。

  • ctx.metadata：formulation、network type、索引和求解器信息。

初始化变量空间#

init_power! 常用参数：

参数

说明

formulation

网络 formulation，例如 :acp, :acr, :ivr, :dcp, :ptdf, :wsoc, :bf_soc

state_variables

状态变量集合，例如 :polar, :rectangular, :ivr, :w_space, :voltage_square

variables

限制要创建的变量族，例如 voltage、generation、branch flow

network_type

:ac 或 :acdc

dc_state_variables

DC 侧状态变量，例如 :voltage, :voltage_square

hvdc_case

PGLIB-HVDC/MatACDC 数据

组件契约#

每个组件声明：

required_networks(component)
required_variables(component)
provided_variables(component)
formulation_short_name(component)
formulation_problem_class(component)

构建顺序为：

  1. 检查网络依赖；

  2. 注册所有组件变量；

  3. 检查变量依赖；

  4. 注册所有约束；

  5. 注册目标项；

  6. 构造总目标函数。

结果恢复#

solution = solve_modular_model!(ctx; solver = :ipopt)
solution.termination_status
solution.primal_status
solution_status(solution)
modular_solution_summary(solution)

底层结果恢复逻辑仍集中在 recover_optimization_result(ctx)，但用户查询建议优先使用更友好的结果 API：

available_solution_variables(solution)
solution_value(solution, :Pg; index = 1, time = 1)
variable_solution(solution, :Pbr_from; time = 1)

bus_solution(solution; time = 1)
generator_solution(solution; time = 1)
branch_solution(solution; time = 1)
storage_solution(solution; time = 1)

power_solution(solution; time = 1)

这些函数既接受 solve_modular_model! 的返回值，也接受 ctx 或 recover_optimization_result(ctx) 的返回值。solution_value 用来查询原始注册变量，默认保留 p.u. 单位；bus_solution、generator_solution、branch_solution 和 storage_solution 会按电力系统语义组织字段，并把有功、无功、视在功率换算为 MW、MVAr、MVA。

常见字段包括：

查询函数

典型字段

bus_solution

vm_pu, va_deg, p_demand_mw, p_generation_mw, load_shed_mw

generator_solution

pg_mw, qg_mvar, on, start, stop, pmin_mw, pmax_mw

branch_solution

p_from_mw, q_from_mvar, p_to_mw, s_from_mva, loading_percent, theta_diff_deg

storage_solution

soc, energy_mwh, charge_mw, discharge_mw, net_power_mw

如果只是在建模阶段调试变量注册，可以继续使用：

has_variable(ctx, :power, :Pg)
get_variable(ctx, :power, :Pg)

模型求解后则推荐使用 solution_value 和 *_solution 系列函数，避免用户手动写 JuMP.value.(get_variable(...))。
